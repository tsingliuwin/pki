# PKI: 个人知识内化系统

**混合架构的高性能知识内化方案**

PKI 是一个探索如何将重型模型微调与轻量化本地推理相结合的系统。它采用“职责分离”架构：在 Python 中进行高精度训练，在 Rust 中进行极速量化推理。

## 🚀 PKI 的哲学

与 RAG（检索增强生成）这种向 Prompt 注入上下文的“外挂”方案不同，PKI 的目标是**将知识内化到模型权重中**。这需要一个严谨的多阶段流水线。

## 🏗 架构与工作流

PKI 的设计在“训练态”和“运行态”之间有着清晰的界限：

### 1. 训练阶段 (重量级 - Python/PyTorch)
- **引擎:** PyTorch + Unsloth / PEFT。
- **权重:** 必须从 **BF16 原始权重**（而非 GGUF）开始。
- **过程:** 对用户生成的数据集进行 LoRA 微调。
- **资源:** 需要约 5GB+ 的显存/内存（不适合低功耗移动设备）。

### 2. 整合阶段 (桥梁)
- **合并 (Merge):** 将 LoRA 适配器权重合并回原始的 BF16 权重。
- **量化 (Quantization):** 使用 `llama.cpp` 将内化后的模型转换为 **GGUF (Q8_0/Q4_K_M)** 格式。

### 3. 推理阶段 (轻量级 - Rust/Candle)
- **引擎:** 基于纯 Rust (`candle`) 的推理实现。
- **权重:** 优化后的 **GGUF** 量化模型。
- **性能:** 冷启动 < 200ms，常驻内存 < 800MB。

## 🛠 模块组成

- **`pki-ingest`**: 纯 Rust 开发的高性能文档解析与 QA 训练集生成工具。
- **`pki-trainer`**: 负责编排 Python 微调环境及模型导出流水线。
- **`pki-engine`**: 针对 GGUF 优化的极速推理底座。
- **`pki-cli`**: 统一的命令行交互入口。

## 🚦 技术纠错 (必读)

1. **GGUF 不可微调**: 量化格式仅供推理。你无法在离散的权重上计算梯度。
2. **Candle 专注于推理**: Rust 的 `candle` 框架在推理端表现优异，但目前尚不适合进行复杂的、稳定的微调训练。
3. **硬件分层**: 你需要一台性能尚可的机器（建议有 GPU）来完成 **内化（训练）** 步骤；但 **使用（推理）** 步骤可以在任何普通电脑上飞速运行。
