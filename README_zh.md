# PKI: 个人知识内化系统

**混合架构的高性能知识内化方案**

PKI 是一个探索如何通过模型权重演化实现真正知识内化的研究项目。**警告：这是一项重型工程任务，对硬件有较高要求。**

## 🚦 诚实的技术现状 (必读)

在运行此项目之前，请务必理解“内化（微调）”与“推理”之间的巨大鸿沟：

1.  **训练不是推理**：尽管在量化后的 Qwen3-0.6B 上推理仅需 <800MB 内存，但**训练**必须从 BF16/FP32 原始权重开始，合计需要 **10GB+ 的显存或内存**。
2.  **GGUF 不可训练**：你无法在 GGUF 文件上计算梯度。训练的起点只能是原始的 Safetensors。
3.  **CPU 训练极其缓慢**：在普通 CPU 上对 0.6B 模型进行 FP32 精度微调，10 个训练步（Steps）可能需要 **数小时** 甚至更久。这绝非实时任务。
4.  **拒绝虚假闭环**：我们不再使用任何“占位符”或“模拟数据”来伪造 Pipeline 的成功。如果硬件不达标，系统将诚实地报告失败。

## 🏗 系统架构

PKI 采用“职责分离”架构，以应对不同的计算密度需求：

### 阶段 1: 摄入 (Rust/GGUF)
- **目标**：解析文档并生成高质量的 QA 训练集。
- **硬件**：任何现代 CPU 均可胜任，内存占用低。

### 阶段 2: 内化 (Python/PyTorch/BF16)
- **目标**：通过真实的 LoRA 微调将知识编码进权重。
- **推荐硬件**：**强烈建议配备 8GB+ 显存的 NVIDIA GPU。**
- **CPU 模式**：被视为“开发者专用/通宵模式”。需要 10GB+ 内存及极长的等待时间。

### 阶段 3: 整合与量化 (技术鸿沟)
- **目标**：合并权重并重新量化为 GGUF 格式供 Rust 引擎使用。
- **现状**：这是一个复杂的步骤，目前需要依赖 `llama.cpp` 工具链或 Python 侧的专用导出算子。

### 阶段 4: 推理 (Rust/GGUF)
- **目标**：使用“已内化”的模型进行极速本地对话。

## 📉 项目状态
当前版本 (v0.2.2) 处于概念验证阶段。它打通了工程链路的骨架，但核心的**知识内化步骤**是一个高成本操作，需要专业级硬件才能在合理时间内看到真实效果。
